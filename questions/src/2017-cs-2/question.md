명령어를 읽어 순차적으로 실행하는 프로세서에 적용되는 기술로, CPU가 한 번에 하나의 명령어만 실행하는
것이 아니라, 하나의 명령어가 실행되는 도중에 다른 명령어 실행을 시작하는 식으로 동시에 여러 개의
명령어를 실행하는 기법이다. 명령어 실행을 위해 필요한 자원을 병렬적으로 사용한다는 것이 특징이며,
전통적으로는 명령어 인출 (instruction fetch) - 명령어 해석 (instruction decode) - 실행 (execute) -
메모리 접근 (memory access) - 레지스터 다시 써넣기 (register write back)의 다섯 단계로 나뉘지만 이를
분화시키려는 다양한 연구가 존재한다. 여러 개의 프로세서를 사용함으로써 병렬화를 이루는 게 아니라,
하나의 프로세서를 사용하면서 명령어 단계에서 병렬화를 한다는 데 의미가 있는, 이 기법의 이름은
무엇인가?
